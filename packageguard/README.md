# 🛡️ PackageGuard

Supply-chain security tool. Scores any package for risk **before** you install it, scans
existing projects for compromised dependencies, and uses a graph neural network to catch
threats hiding *inside* a dependency.

Two faces, one engine:
- **CLI** — `packageguard check <pkg>` / `packageguard scan <path>` / `packageguard graph <pkg>`
- **HUD dashboard** — `packageguard serve` → home page, live CHECK/SCAN/GRAPH tools, and a
  supply-chain security awareness page, all in a tactical browser console

## How it works

- **Per-package model** — trained **XGBoost** over 8 metadata features (typosquat similarity,
  install scripts, author age, publish timing, dependency count, release history, description
  quality, maintainer count) with **SHAP** attributions; transparent heuristic fallback offline.
- **Graph model** — a **GraphSAGE** GNN scores a package via its dependency-graph neighbourhood,
  catching *poisoned chains* per-package scoring misses. Validated on a held-out-unknown
  benchmark: 89% recall / 0% false alarms on real threats never seen during training, vs. 0%
  recall for a deterministic database lookup. Combined via a transparent additive log-odds
  formula (see `training/RESULTS.md` for why a trained stacking combiner was tried twice and
  not shipped).
- **Scan** — pure `package-lock.json` parsing (no code execution) + a bundled known-malware DB,
  with historical-incident notices for packages whose current version is safe but had a past
  incident.

## Quick start

```bash
python -m venv .venv
.venv\Scripts\activate            # Windows  (source .venv/bin/activate on macOS/Linux)
pip install -e ".[ml,gnn]"

packageguard check co1ors         # typosquat  -> DO NOT INSTALL
packageguard check express        # clean      -> LIKELY SAFE
packageguard scan ./sample        # project with malware
packageguard graph safe-wrapper   # poisoned-chain demo (GNN)
packageguard serve                # HUD dashboard
```

Models are regenerated by the scripts in `training/` (not committed). Without them the tool runs
on its heuristic fallback. See `training/RESULTS.md` for the full evaluation history.

## Layout

| Path | Role |
| ---- | ---- |
| `src/packageguard/core/` | Engine — features, scoring (XGBoost/heuristic), GNN, combiner, lockfile, remediation. |
| `src/packageguard/cli.py` | Typer CLI: `check`, `scan`, `graph`, `serve`. |
| `src/packageguard/api/` | FastAPI backend; serves the web UI. |
| `src/packageguard/web/` | HUD dashboard — home, CHECK, SCAN, GRAPH, and awareness pages (vanilla HTML/CSS/JS + SVG dependency graph). |
| `src/packageguard/data/` | Bundled known-malware DB, curated poisoned-chain demos, popularity list. |
| `training/` | Data pipeline + model training (XGBoost + GraphSAGE) + the held-out-unknown benchmark. |
| `tests/` | Unit + edge-case tests. |

## Reproducing the models

```bash
python training/build_dataset.py && python training/train_xgboost.py
python training/build_graph_dataset.py && python training/train_gnn.py
python training/build_heldout_benchmark.py && python training/evaluate_heldout_benchmark.py
```
